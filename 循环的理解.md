# `if q and q[0] <= i-k:`，`while q and nums[q[-1]]<nums[i]:`，这两个怎么理解？

这是**整道题里最核心的两句判断**。
下面**严格零基础、人话、不偷懒**解释，而且**一条一句拆开讲清楚“为什么必须这样写”**。

---

# 一、先把两句话原样放出来

```python
if q and q[0] <= i - k:
```

```python
while q and nums[q[-1]] < nums[i]:
```

这两句**一个管“过期”**，**一个管“大小关系”**。
顺序也**绝对不能乱**。

---

# 二、第一句：

## `if q and q[0] <= i - k:`

### 👉 这句只做一件事：

> **把已经“滑出窗口”的下标删掉**

---

## （1）一句一句拆

### ① `q`

* `q` 是队列
* 在 `if` 里，`q` 的意思是：
  👉 **队列不是空的**

---

### ② `and`

* `and` 表示：
  👉 **左右两个条件都要为真**

---

### ③ `q[0]`

* 队头的下标
* 代表：
  👉 **当前窗口里“最老”的候选值**

---

### ④ `i - k`

这是关键。

#### 当前窗口的范围是：

```
[i - k + 1 , i]
```

所以：

* `i - k`
  👉 **窗口左边界的前一个位置**

---

### ⑤ `q[0] <= i - k`

含义是：

> 队头下标 **已经跑到窗口外面去了**

---

## （2）用真实数字看

```
nums = [1,3,-1,-3,5]
k = 3
i = 4
```

当前窗口是：

```
下标: [2,3,4]
```

如果：

```
q[0] = 1
```

那就说明：

* 下标 1 不在窗口内
* **这个数再大也没资格当最大值**

---

## （3）为什么用 `if`，不是 `while`？

因为：

* 每次窗口只右移一格
* **最多只有一个下标会过期**

---

## （4）一句人话总结

> **“这个最大值如果已经不在窗口里了，立刻扔掉”**

---

# 三、第二句：

## `while q and nums[q[-1]] < nums[i]:`

### 👉 这句只做一件事：

> **把比当前数小的“废物候选值”全部删掉**

---

## （1）一句一句拆

### ① `q`

* 队列非空

---

### ② `nums[q[-1]]`

* `q[-1]`：队尾下标
* `nums[q[-1]]`：队尾对应的数值

---

### ③ `nums[i]`

* 当前新进窗口的数

---

### ④ `nums[q[-1]] < nums[i]`

意思是：

> **队尾这个数，比新来的数还小**

---

## （2）为什么要删队尾？

因为：

* 队尾是 **最小的候选值**
* 新数比它大
* 而且新数更靠右（活得更久）

结论：

> 队尾这个数 **永远不可能成为最大值**

---

## （3）为什么是 `while`，不是 `if`？

因为：

* 新数可能比**好几个旧数都大**
* 要一直删，直到：

  * 队列空了
  * 或者队尾比新数大

---

## （4）用真实例子看

```
q 中下标: [1, 2, 3]
对应 nums: [3, -1, -3]

新数 nums[i] = 5
```

执行过程：

```
-3 < 5 → 删
-1 < 5 → 删
3  < 5 → 删
```

最后：

```
q = []
```

---

## （5）一句人话总结

> **“新来的更厉害，后面的都让位”**

---

# 四、两句放一起的“终极理解”

### 第一句（if）：

> **清理窗口外的**

### 第二句（while）：

> **清理比新数小的**

顺序一定是：

```
先清理过期的
再清理没资格的
最后加入自己
```

---

# 五、极度压缩版（背诵用）

> `q[0] <= i-k`：
> **老的走**

> `nums[q[-1]] < nums[i]`：
> **小的滚**

---

# 六、用一句话彻底记住

> **窗口只留“还在窗口里、而且有希望当老大”的人**
