```python
问题：

q = [1, 2, 3]

nums = [1, 3, -1, -3, 5]

是不是因为deque要求下标不能是0，只能从1开始，所以q[0]=1，而nums[q[0]]=nums[1]=3？
```

不是的，这个理解**完全不对**，而且这是一个**非常典型、必须立刻纠正的误区**。
下面把这件事**一次性、彻底讲清楚**，避免以后反复踩坑。

---

## 一、先给结论（非常重要）

> ❌ **deque 并不要求下标从 1 开始**
> 
> ❌ **deque 和 list 一样，下标永远从 0 开始**

> ✅ **`q[0] = 1` 的原因只有一个：
> 
> 因为队列里第一个存进去的“元素本身就是 1”**

---

## 二、把这段代码拆到“不能再拆”为止

### 已知：

```python
q = [1, 2, 3]
nums = [1, 3, -1, -3, 5]
```

---

## 三、先弄清楚：`q` 里面到底存的是什么？

在这道题中，**约定**是：

> **q 里存的是 nums 的下标**

所以：

| q 里的元素 | 含义         |
| ------ | ---------- |
| 1      | nums 的下标 1 |
| 2      | nums 的下标 2 |
| 3      | nums 的下标 3 |

---

## 四、关键点来了：`q[0]` 到底是什么意思？

### 1️⃣ Python 的统一规则（不管 list 还是 deque）

```python
某个容器[0]
```

含义永远是：

> **取“第一个位置”上的元素**

---

### 2️⃣ 套用到当前例子

```python
q = [1, 2, 3]
```

位置和元素是：

| 位置（下标） | 元素值 |
| ------ | --- |
| 0      | 1   |
| 1      | 2   |
| 2      | 3   |

所以：

```python
q[0] == 1
```

⚠️ **这里的 0 是“q 的位置”，不是 nums 的下标**

---

## 五、为什么还要写 `nums[q[0]]`？

因为：

* `q[0]` 得到的是 **nums 的下标**
* 要拿到真正的数据，必须再去 `nums` 里取一次

```python
nums[q[0]] = nums[1] = 3
```

---

## 六、把整个过程用一句“绝对不混”的话说清楚

> **`q[0]` 是：
> 队列里的第一个元素
> 而这个元素“恰好是 nums 的一个下标”**

---

## 七、重点澄清：deque 和下标起点毫无关系

下面这个例子可以直接打碎误解：

```python
from collections import deque
q = deque()
q.append(0)
q.append(1)
q.append(2)
print(q)
```

结果是：

```
[0, 1, 2]
```

说明：

* deque **完全可以存 0**
* deque **不限制下标值**
* 0、1、100、-5 都可以存

---

## 八、为什么你会产生“下标从 1 开始”的错觉？

原因只有一个：

> **队列里“刚好存了 1”
> 而不是“必须从 1 开始”**

这是一个**巧合，不是规则**。

---

## 九、用一句话把误解彻底封死（必须背）

> **deque 不管下标是什么；
> q[0] 只是取第一个元素；
> 元素是不是 1，完全由代码决定**

---

## 十、再给一个对照，防止以后再混

### 错误理解（❌）

> deque 要求下标从 1 开始

### 正确理解（✅）

> deque 只是装东西
> 装的是下标
> 下标从 0 还是 1，完全是代码自己决定的

---

## 十一、终极一句话总结

> **0 是“位置”，1 是“内容”，两者不是一回事**
