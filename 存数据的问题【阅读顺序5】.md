```python
问题：
            if i >= k-1:
                result.append(nums[q[0]])
            # if i >= k:
            #     result.append(nums[i])

下面注释的为何不对？

为啥大于`k-1`就开始记录最大值了呢？不应该大于`k`才行吗，因为大于`k`才形成窗口。

为何添加的是`nums[q[0]]`，`q[0]`不是下标吗？我们要存的不是数据吗？因此为何添加的不是`nums[i]`，但是`nums[i]`也是下标。
```

这个问题**一次性问中了三个“初学者最容易混在一起的点”**。
下面**严格拆成三块**，每一块都从“下标—窗口—数据”三个层面讲清楚，保证逻辑完全闭合。

---

# 一、为什么是 `i >= k - 1`，而不是 `i >= k`？

## 1️⃣ 先明确一件事（非常关键）

> **窗口是用“下标”来数的，不是用“元素个数的感觉”来猜的**

---

## 2️⃣ 窗口大小是 `k`，下标从 `0` 开始

假设：

```
k = 3
```

### 第一个合法窗口应该是哪些下标？

```
[0, 1, 2]
```

注意：

* 一共 3 个数
* **最右边的下标是 2**

---

## 3️⃣ 用公式算“窗口右边界”

窗口右边界下标 =

```
k - 1
```

因为：

* 第 1 个数：下标 0
* 第 2 个数：下标 1
* 第 3 个数：下标 2 = k - 1

---

## 4️⃣ 所以什么时候“窗口第一次形成”？

当：

```
i = k - 1
```

而不是 `k`

---

## 5️⃣ 用具体数值对比（一眼就明白）

### k = 3

| i | 已看到的下标  | 个数 | 是否形成窗口 |
| - | ------- | -- | ------ |
| 0 | [0]     | 1  | ❌      |
| 1 | [0,1]   | 2  | ❌      |
| 2 | [0,1,2] | 3  | ✅      |
| 3 | [1,2,3] | 3  | ✅      |

👉 **第一次形成窗口是在 `i = 2 = k - 1`**

---

## 6️⃣ 所以这句是完全正确的

```python
if i >= k - 1:
```

而如果写成：

```python
if i >= k:
```

❌ 会**漏掉第一个窗口**

---

# 二、为什么要 `result.append(nums[q[0]])`？

这是**本题的灵魂问题**。

---

## 1️⃣ 再次强调（必须记死）

> **队列 `q` 里存的是“下标”，不是数据**

例如：

```
q = [1, 2, 3]
nums = [1, 3, -1, -3, 5]
```

含义是：

```
下标 1 → 数值 3
下标 2 → 数值 -1
下标 3 → 数值 -3
```

---

## 2️⃣ `q[0]` 是什么？

```python
q[0]
```

得到的是：

```
1   （下标）
```

---

## 3️⃣ `nums[q[0]]` 才是什么？

```python
nums[q[0]] = nums[1] = 3
```

👉 **这才是当前窗口的最大值**

---

## 4️⃣ 为什么“最大值一定在 q[0]”？

因为前面代码**强制维护了一个规则**：

```python
while q and nums[q[-1]] < nums[i]:
    q.pop()
```

这条规则保证：

```
nums[q[0]] ≥ nums[q[1]] ≥ nums[q[2]] ...
```

所以：

> **队头永远是当前窗口的最大值下标**

---

## 5️⃣ 所以必须写成

```python
result.append(nums[q[0]])
```

而不是：

```python
result.append(q[0])      # ❌ 下标，不是数据
```

---

# 三、为什么不能写成 `result.append(nums[i])`？

这是**第三个误区**。

---

## 1️⃣ `nums[i]` 是什么？

```python
nums[i]
```

含义是：

> **当前刚进窗口的那个数**

---

## 2️⃣ 当前数 ≠ 最大值（大多数时候）

举例：

```
nums = [1, 3, -1]
k = 3
```

当：

```
i = 2
```

* `nums[i] = -1`
* 但窗口最大值是 `3`

如果写：

```python
result.append(nums[i])
```

得到的是：

```
-1 ❌
```

---

## 3️⃣ 再举一个更明显的例子

```
nums = [5, 4, 3, 2, 1]
k = 3
```

第一个窗口 `[5,4,3]`：

* `nums[i] = 3`
* 最大值 = 5

❌ 完全不一样

---

## 4️⃣ 本题要的是

> **窗口里的最大值**
>
> 而不是
> **“刚进来的值”**

---

# 四、把三个问题合成一句“绝对正确”的逻辑

> * `i >= k - 1`：
>   **窗口第一次形成**
>
> * `q[0]`：
>   **最大值的下标**
>
> * `nums[q[0]]`：
>   **最大值本身**

---

# 五、把错误写法逐条“判死刑”

```python
# if i >= k:
```

❌ 漏第一个窗口

```python
# result.append(nums[i])
```

❌ 记录的是新数，不是最大值

```python
# result.append(q[0])
```

❌ 记录的是下标，不是数据

---

# 六、终极背诵版（一定要记）

> **窗口满：i ≥ k − 1**

> **最大值：nums[q[0]]**
