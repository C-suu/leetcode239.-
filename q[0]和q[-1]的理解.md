# q[0],q[-1]是固定用法吗？啥意思?

这是一个**非常关键、问得非常准的问题**。
结论先给出，然后再从**零基础、人话层面**慢慢拆开。

---

## 一、直接结论（先记住）

> **`q[0]` 和 `q[-1]` 不是算法规定的“固定写法”**
> **而是 Python 里“取队列元素”的通用语法**

在这道题里：

* `q[0]`
  👉 **队头（最左边）的下标 = 当前窗口最大值的位置**

* `q[-1]`
  👉 **队尾（最右边）的下标 = 最新、最小的候选值**

---

## 二、`q` 到底是什么？

```python
from collections import deque
q = deque()
```

### `deque` 是什么？

* 全名：**double ended queue（双端队列）**
* 特点：

  * 左边能进、能出
  * 右边也能进、能出

可以把它想象成：

```
左边 ←  [   队列里的东西   ]  → 右边
```

---

## 三、`q[0]` 是啥意思？（彻底讲清楚）

### 1️⃣ Python 里下标的基本规则

对任何“像列表一样的东西”：

```python
x = [10, 20, 30]
```

* `x[0]` → 第 1 个
* `x[1]` → 第 2 个
* `x[2]` → 第 3 个

---

### 2️⃣ 负数下标是什么意思？

Python 规定：

* `-1` → **最后一个**
* `-2` → 倒数第二个

```python
x = [10, 20, 30]
x[-1] = 30
x[-2] = 20
```

---

### 3️⃣ `deque` 也遵守同样规则

```python
q = deque([1, 4, 6])
```

```
下标:   0   1   2
元素:   1   4   6
```

* `q[0]`  → 1（队头）
* `q[-1]` → 6（队尾）

✅ **不是专门给滑动窗口准备的**
✅ **是 Python 的通用下标语法**

---

## 四、为什么滑动窗口里“必须用”`q[0]` 和 `q[-1]`？

不是“必须”，而是**最合理**

---

### （1）为什么最大值在 `q[0]`？

队列的规则是：

> **队列里的下标，对应的数值是从大到小排的**

例子：

```
nums = [1, 3, -1, -3, 5]
q = [1, 2, 3]
```

对应数值：

```
下标:    1    2    3
数值:    3   -1   -3
```

最大值一定在最左边：

```python
nums[q[0]]  # = nums[1] = 3
```

---

### （2）为什么比较用 `q[-1]`？

新数 `nums[i]` 进来时：

```python
while q and nums[q[-1]] < nums[i]:
    q.pop()
```

意思是：

> “只要队尾的数比新数小，就删掉”

因为：

* 队尾是 **最小的候选值**
* 一旦有更大的新数进来
* 队尾 **永远不可能再成为最大值**

---

## 五、能不能不用 `q[0]` 和 `q[-1]`？

### 能，但没必要

可以写成：

```python
q[0]        # 队头
q[len(q)-1]# 队尾
```

但：

* 更啰嗦
* 更容易写错
* 可读性更差

所以大家**约定俗成**写成：

```python
q[0]
q[-1]
```

---

## 六、把话说“非常直白”的一句话版本

> `q[0]`：
> **“当前窗口里最大的那个数的下标”**

> `q[-1]`：
> **“当前窗口里最小、最容易被挤掉的那个下标”**

---

## 七、背诵用一句话（考试/面试用）

> **队头取最大，队尾做比较**
