# leetcode239. 滑动窗口最大值：给一个整数数组 nums，有一个大小为 k 的滑动窗口从数组的最左侧移动到数组的最右侧。只可以看到在滑动窗口内的 k 个数字。滑动窗口每次只向右移动一位。返回：滑动窗口中的最大值。

下面**严格按照给定顺序**来说明「239. 滑动窗口最大值」。
内容会**非常非常细**，假设是**完全零基础**，并且**不偷懒**。

---

# （0）【纯人话、不写代码】讲清楚解题思路（零基础版）

---

## 一、题目在说什么（先翻译成人话）

有一个整数数组，比如：

```
[1, 3, -1, -3, 5, 3, 6, 7]
```

还有一个数字 `k`，比如 `k = 3`。

### 什么是“滑动窗口”？

* 窗口 = 一个“框”
* 框的**大小固定是 k**
* 框从数组**最左边**开始
* 每次只向右移动 **1 个位置**
* 每次只能看到框里的 `k` 个数字

目标只有一个：

> **每次窗口停下来时，记下窗口里的最大值**

---

## 二、最笨的方法（但能理解）

比如第一个窗口：

```
[1, 3, -1]   → 最大值是 3
```

第二个窗口：

```
[3, -1, -3] → 最大值是 3
```

第三个窗口：

```
[-1, -3, 5] → 最大值是 5
```

……

如果每次都**重新把窗口里的 k 个数全部比较一遍**：

* 每个窗口要比较 `k` 次
* 一共有 `n` 个窗口
* 总复杂度是 `O(n × k)` → **会超时**

---

## 三、聪明的核心思想（关键！）

### 目标：

**不用每次重新比较 k 个数**

### 关键观察：

> **窗口右移时，只发生两件事**

1. 左边有一个数“离开窗口”
2. 右边有一个新数“进入窗口”

---

## 四、用一个“特殊队列”来帮忙（单调队列）

这个队列有三个规则：

### 规则 1：队列里只存「下标」，不直接存数值

原因：

* 要判断某个数**是否已经滑出窗口**

---

### 规则 2：队列里的数值 **从大到小排列**

意思是：

```
队头 → 当前窗口最大值
```

---

### 规则 3：新数字进来时

* 如果新数字 **比队尾对应的数字大**
* 那么队尾的数字 **永远不可能再成为最大值**
* 直接从队尾删除

---

## 五、为什么这样一定正确？

* 队头：**窗口中最大的数**
* 队列中保存的都是：

  * 还在窗口里的
  * 有资格成为最大值的
* 每个元素 **只进队一次，只出队一次**
* 时间复杂度是 `O(n)`

---

# （1）Python3 标准答案（每一行都有中文注释）

```python
from collections import deque   # 导入双端队列工具

class Solution:
    def maxSlidingWindow(self, nums, k):
        q = deque()    # 创建一个空的双端队列，用来存“下标”。左边是队头q[0]，右边是队尾q[-1]。
        res = []       # 创建结果列表，用来存每个窗口的最大值

        for i in range(len(nums)):    # i 从 0 遍历到 nums 的最后一个下标

            # 如果队头的下标已经不在窗口范围内，弹出
            if q and q[0] <= i - k:
                q.popleft() # 从左边删

            # 如果当前数比队尾对应的数大，队尾出队
            while q and nums[q[-1]] < nums[i]:
            # 这句代码的真实含义是：只要新来的数更大，就把后面所有比它小的都踢掉
                q.pop() # 从右边删
            q.append(i)   # 把当前下标加入队列，这个是人为维护deque()的排序。

            # 当窗口形成之后，开始记录最大值
            if i >= k - 1:
                res.append(nums[q[0]])   # 队头对应的数就是当前窗口最大值

        return res                           
```

# （4）关键公式 / 表达式来源解释

---

## ① `q[0] <= i - k`

来源：

* 当前窗口范围是 `[i-k+1 , i]`
* 小于 `i-k+1` 的下标已经不在窗口
* 等价写法：`<= i-k`

---

## ② 单调递减队列

来源：

* 最大值只可能出现在“还没被更大值挡住”的元素中
* 被更大的新元素挡住的旧元素，永远不可能再成为最大值

---

# （6）【背诵法则】——死记也能写出来

---

### 一句话口诀

> **下标入队，过期出队，小的删掉，头部最大**

---

### 四步背诵法

1️⃣ 队列里 **只存下标**
2️⃣ 队列保持 **从大到小**
3️⃣ 窗口外的 **立即删除**
4️⃣ 窗口成型后 **取队头**

---

```python
for i in range(n):
    删过期
    删小的
    加自己
    记答案
```
